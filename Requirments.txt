7. Software Requirements Specification (SRS)
1. Functional Requirements:

| ID    | Requirement                                                                | Priority |
| ----- | -------------------------------------------------------------------------- | -------- |
| FR-01 | System shall support Arabic and local dialect NLP                          | High     |
| FR-02 | System shall classify transactions as personal/business with >85% accuracy | High     |
| FR-03 | System shall calculate Funding Readiness Score (0-100)                     | High     |
| FR-04 | System shall generate explainability reports in Arabic                     | High     |
| FR-05 | System shall create personalized improvement roadmaps                      | High     |
| FR-06 | System shall support re-evaluation after user actions                      | High     |
| FR-07 | System shall integrate with bank APIs via Open Banking                     | High     |
| FR-08 | System shall enforce user consent for data access                          | High     |
| FR-09 | System shall detect anomalous transactions                                 | Medium   |
| FR-10 | System shall provide counterfactual explanations                           | Medium   |
| FR-11 | System shall benchmark user against sector averages                        | Medium   |
| FR-12 | System shall support multiple bank policies                                | Medium   |



2. Non-Functional Requirements:
| ID     | Requirement              | Target                              |
| ------ | ------------------------ | ----------------------------------- |
| NFR-01 | Assessment response time | < 3 seconds                         |
| NFR-02 | Chat response time       | < 1 second                          |
| NFR-03 | System availability      | 99.9%                               |
| NFR-04 | Data encryption          | AES-256 at rest, TLS 1.3 in transit |
| NFR-05 | Concurrent users         | 10,000+                             |
| NFR-06 | Data retention           | 7 years (banking compliance)        |
| NFR-07 | Accessibility            | WCAG 2.1 AA                         |
| NFR-08 | Audit trail              | Immutable logs                      |



3. Security Requirements:

All PII encrypted at field level
Bank account numbers hashed (SHA-256 + salt)
RBAC (Role-Based Access Control)
MFA for bank staff access
API rate limiting: 100 req/min per user
Data anonymization for ML training
